# Spring + Unity 실시간 멀티플레이 데모

> **기존 문서 협업 WebSocket 서버(Co-Write)의 통신 구조를 재활용해서,  
> Unity 3D 템플릿 위에 실시간 멀티플레이를 구현한 도전 과제입니다.**

---

## 1. 프로젝트 소개

이 프로젝트는 **Spring 기반 WebSocket 서버**와 **Unity 3D 클라이언트**를 이용해  
여러 사용자가 동시에 같은 게임 맵에 접속해 함께 움직일 수 있는  
**간단한 실시간 멀티플레이 데모**를 구현한 것입니다.

원래 저는 문서 협업 서비스(Co-Write)를 개발하면서  
- STOMP/WebSocket 기반 실시간 편집  
- 브로드캐스트, 방 개념, 이벤트 기반 메시징  

을 이미 경험했는데,  
이번 자유 주제에서는 이 경험을 **“게임 네트워크”라는 새로운 영역으로 확장**해 보고자 했습니다.

---

## 2. 도전 주제

> **웹 백엔드(Spring, WebSocket) 개발자가  
> 직접 만든 실시간 통신 모듈을 이용해  
> Unity 3D 멀티플레이 게임을 구현해보기**


- 지금까지는 **문서 협업** 같은 텍스트 기반 실시간 서비스만 구현해 봤습니다.
- **게임 서버**는 전용 프레임워크(Photon 등)를 많이 사용하는 영역이지만, Spring을 이용하며 발생하는 문제점들을 파악해 보고 싶습니다.
- 완전히 새로운 스택으로 시작하지 않고,  **기존 Co-Write 의 Kafka, Websocket을 이용한 서버 모듈을 “게임”에 재활용**해 보는 시도가 의미 있다고 판단했습니다.

---

## 3. 주요 기능

### ✅ 기본 기능

- 게임을 시작하면 여러 클라이언트가 **같은 방(Room)** 에 접속됩니다.
- 각 클라이언트의 **플레이어 위치/회전 정보**를 서버로 전송시킵니다.
- 서버는 해당 방에 접속한 모든 플레이어에게  
  **현재 룸의 전체 플레이어 상태를 브로드캐스트**합니다.
- Unity 3D 템플릿 씬에서
  - 내 캐릭터는 로컬 입력(WASD 등)으로 움직이고
  - 다른 플레이어 캐릭터는 **서버에서 받은 좌표**로 동기화시킵니다.

### 메시지 타입 예시
- `INPUT` : 입력/이동 정보 전송
